Asynchronous JavaScript and APIs

1. Promises

A **Promise** is a JavaScript object that represents the eventual completion or failure of an asynchronous operation. It's a cleaner way to handle asynchronous code, avoiding "callback hell." A Promise can be in one of three states: pending, fulfilled, or rejected.

const myPromise = new Promise((resolve, reject) => {
  const success = true;
  if (success) {
    resolve('Operation was successful!');
  } else {
    reject('Operation failed.');
  }
});
myPromise.then(result => console.log(result));

2. The `async/await` Syntax

The `async/await` syntax, built on top of Promises, provides a more readable and synchronous-looking way to write asynchronous code. The `async` keyword makes a function return a Promise, and `await` pauses the function's execution until the Promise is resolved.

async function fetchData() {
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();
  return data;
}
fetchData().then(data => console.log(data));

3. What are APIs?

An **API (Application Programming Interface)** is a set of rules that allows different applications to communicate with each other. In web development, we often use REST APIs to request and receive data from a server, like fetching a list of products or a user's profile information. We use the `fetch()` API in JavaScript to make these requests.

fetch('https://api.github.com/users/octocat')
  .then(response => response.json())
  .then(data => console.log(data.login));
