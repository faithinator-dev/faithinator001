Creating Elements and Handling Events

1. Creating and Appending Elements

You can't just change existing elements; you can also create new ones from scratch. The `document.createElement()` method creates a new element in memory, and then you can use methods like `appendChild()` to insert it into the DOM tree, making it visible on the page.

// Create a new paragraph element
const newParagraph = document.createElement('p');

// Set its text content
newParagraph.textContent = "I'm a new paragraph!";

// Find a container to append it to
const container = document.querySelector('#container');

// Add the new element to the container
container.appendChild(newParagraph);

2. Modifying Attributes and Classes

To have full control over elements, you need to be able to change their attributes and CSS classes. The `classList` property provides a simple way to add, remove, or toggle classes on an element, and `setAttribute()` can be used to set any HTML attribute, like `src` for an image or `href` for a link.

const image = document.querySelector('img');

// Set an attribute
image.setAttribute('src', 'path/to/new-image.jpg');

const card = document.querySelector('.card');

// Add a class
card.classList.add('active');

// Remove a class
card.classList.remove('inactive');

3. Event Listeners

Web applications are all about reacting to user input. An **event listener** is a function that waits for an event to occur on a specific element, such as a click, a key press, or a form submission. When the event happens, the listener's function is executed, allowing you to run code in response to user actions.

const myButton = document.querySelector('button');

myButton.addEventListener('click', () => {
  console.log('Button was clicked!');
});
