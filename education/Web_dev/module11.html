Handling Asynchronous Operations

1. The Problem with Synchronous Code

So far, all the JavaScript you've written has been **synchronous**. This means each line of code is executed one after another. This works fine for simple tasks, but what happens when you need to fetch data from a server? That could take seconds, and if your code just sits there and waits, your entire web page would freeze. This is why we need **asynchronous** code.

// Synchronous example
console.log('Task 1');
// Imagine this line takes 5 seconds to complete, freezing the browser.
// The browser will wait here and not render anything else.
console.log('Task 2');

2. Promises

A **Promise** is an object representing the eventual completion or failure of an asynchronous operation. A Promise is in one of three states: pending, fulfilled, or rejected. The `then()` method is used to handle the fulfilled state, and `catch()` handles the rejected state.

// An example of a Promise
const myPromise = new Promise((resolve, reject) => {
    // Simulating an asynchronous operation
    setTimeout(() => {
        resolve('Operation successful!');
    }, 2000);
});

myPromise.then(result => {
    console.log(result); // Logs 'Operation successful!' after 2 seconds
}).catch(error => {
    console.error(error);
});

3. Async/Await

**Async/Await** is a modern syntax for working with Promises that makes asynchronous code look and behave more like synchronous code, which is much easier to read and write. The `async` keyword defines a function that returns a Promise, and `await` pauses the execution of an `async` function until a Promise is resolved.

async function fetchData() {
    try {
        // await pauses the function until the Promise is resolved
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Error fetching data:', error);
    }
}
